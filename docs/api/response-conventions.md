# Соглашения об ответах

Формы ответов, общие для всего приложения. Документирует то, что иначе пришлось бы
выводить из кода для каждой страницы.

## Три типа ответа

| Тип          | Что это                                                                                            | Когда                                               |
| ------------ | -------------------------------------------------------------------------------------------------- | --------------------------------------------------- |
| **Inertia**  | объект страницы `Inertia::render('Xxx/Index', [...])` (HTML при первом заходе, JSON при навигации) | GET-страницы разделов                               |
| **Redirect** | `RedirectResponse` (`redirect()->route(...)` или `back()`) с flash и/или ошибками валидации        | мутации (`store`/`update`/`destroy`/…)              |
| **JSON**     | `response()->json(...)` — настоящий XHR-ответ                                                      | 4 ручки, см. [json-endpoints.md](json-endpoints.md) |

Мутации **не** возвращают JSON — они редиректят, а результат показывается через
flash-тост на следующей странице.

## Shared-пропсы (на каждый Inertia-ответ)

Источник — `App\Http\Middleware\HandleInertiaRequests::share()`. Доступны во всех
Vue-страницах через `usePage().props`:

| Проп          | Содержимое                                                                                                                 |
| ------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `auth.user`   | текущий пользователь (или `null`)                                                                                          |
| `auth.can`    | плоский список **имён прав** пользователя; читается хелпером `can(perm)` (`resources/js/lib/can.js`) для условного рендера |
| `counts`      | ленивые агрегаты для бейджей сайдбара/колокольчика                                                                         |
| `flash`       | flash-сообщения (`success`/`error`/`warning`/`info`) → тосты                                                               |
| `bot.enabled` | включён ли модуль бота (`config('bot.enabled')`) — гейтит пункт сайдбара                                                   |
| `appName`     | имя приложения (из настроек)                                                                                               |

Любая глобально нужная во Vue информация добавляется сюда, а не прокидывается в
каждый контроллер.

## Конверт пагинации

Списочные страницы (`/users`, `/roles`, `/media`, `/activity-log`, …) отдают
**стандартный пагинатор Laravel** внутри Inertia-пропа. Форма (поля, которые
использует фронт):

```json
{
  "data": [ /* строки текущей страницы */ ],
  "current_page": 1,
  "last_page": 5,
  "per_page": 10,
  "total": 47,
  "links": [ { "url": "...", "label": "1", "active": true }, ... ],
  "from": 1,
  "to": 10
}
```

Размеры страниц: пользователи — 10, роли — 15, медиа — 15, журнал — 30.
Все списки используют `->withQueryString()`, поэтому фильтры/сортировка переживают
переход по страницам.

Сортировка управляется стратегиями `App\Http\Sorts\*` с whitelist полей (защита от
инъекции через `?sort=`); рядом с пагинатором приходят `currentSort` и
`currentDirection`.

## Инфраструктурные маршруты

| Метод | URI   | Что                                                                                                                                                                                  |
| ----- | ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| GET   | `/up` | health-check Laravel (`bootstrap/app.php`, `health: '/up'`). Возвращает 200, если приложение поднялось. Использовать для liveness/readiness-проб (Docker healthcheck, балансировщик) |
| GET   | `/`   | редирект на `/admin`                                                                                                                                                                 |
| —     | 404   | не-admin, не-JSON 404 редиректит на `/admin` (`bootstrap/app.php`)                                                                                                                   |
